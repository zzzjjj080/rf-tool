CORDIC

IQ信号 → 事前回転チェック → 直角三角形の積み上げ → 振幅・位相
なぜCORDICはすごいのか？

ある瞬間の信号から I = A·cos(θ) と Q = A·sin(θ) の たった2つの数値をサンプリングするだけで、振幅 A と 位相 θ の 両方を完全に復元できます。これが IQデモジュレーション の本質です。

CORDICは1959年Jack Volder考案。ハードウェアに乗算器がなかった時代に生まれた天才的アルゴリズムで、 GPS・ソフトウェア無線・FPGAなど現代の信号処理でも現役。直角三角形をビットシフトで積み重ねることで 任意の角度への回転を実現します。

// CORDICの動作原理

CORDICベクタリングモードはベクトル (I, Q) を出発点とし、y成分がゼロになるまで回転させるアルゴリズムです。 毎ステップ 「y > 0なら時計回り、y < 0なら反時計回り」 — yの符号だけで決まります。 累積回転角から θ が、最終ベクトル長から 振幅A が同時に確定。乗算ゼロ、ビットシフトと加減算だけで動作します。
① 収束範囲と事前回転（I符号チェック）
CORDICベクタリングが直接収束できる範囲は x成分 > 0（象限I/IV）のとき。
I = A·cos(θ) < 0（象限II/III）なら、まず ±90° 事前回転で x成分を正に直します。
象限II (I<0, Q≥0) → (I,Q) を −90° 回転 → (Q, −I) からCORDIC開始
象限III (I<0, Q<0) → (I,Q) を +90° 回転 → (−Q, I) からCORDIC開始
θ は見ていない — I の符号だけで判定できます。
② 各ステップの計算 — y符号のみ参照
ステップ i の回転角 = arctan(2⁻ⁱ)（45°→26.6°→14.0°…）。
方向 d は y 成分の符号だけで決まります：
y > 0 → d = −1（時計回り、y を下げる）
y < 0 → d = +1（反時計回り、y を上げる）
x' = x − d·y·2⁻ⁱ　y' = y + d·x·2⁻ⁱ（乗算ゼロ：ビットシフト＋加減算のみ）
累積回転角 accum = Σ d·arctan(2⁻ⁱ) → 最終 θ = −accum として確定。
③ 振幅の取り出し — 最終斜辺の長さ × K
各ステップで直角三角形を重ねるたびに 新ベクトル = 斜辺 になり、ベクトル長が伸び続けます。
y が 0 に収束した最終状態では x成分 ≈ |v|（ベクトルがほぼx軸上）。
n ステップ後の |v| は元の振幅 A より 1/K ≈ 1.647 倍大きい（K ≈ 0.6073 は固定値）。
振幅A = x_final × K ≈ |v| × K　これだけで元の振幅が確定します。
緑の点線が「斜辺の先端 → K倍縮め → 振幅A」を示しています。
最終行「✓ y≈0、振幅確定」でθとAの両方が同時に求まります。

// なぜ K で補正するのか — 斜辺の伸びを直す

①
CORDICの回転は「疑似回転」
本来の純粋な回転はベクトルの長さを変えませんが、CORDICのステップは
x' = x − d·y·2⁻ⁱ、y' = y + d·x·2⁻ⁱ
この計算は回転ではなく 直角三角形を新たに重ねる 操作です。
②
ステップごとに斜辺が底辺より長くなる
直角三角形では必ず 斜辺 > 底辺。ステップ i での伸び率は：
√(1 + 2⁻²ⁱ) 倍
例：ステップ 0 → 底辺 1 に対して 2⁻⁰=1 の直角辺 → 斜辺 = √2 ≈ 1.414 倍
  ステップ 1 → 底辺 1 に対して 2⁻¹=0.5 の直角辺 → 斜辺 = √1.25 ≈ 1.118 倍
③
n ステップ後の合計伸び率は固定値
全ステップの伸び率を掛け合わせると：
1/K = √2 × √1.25 × √1.0625 × … ≈ 1.6468
この値は ステップ数が決まれば一定（入力する角度 θ や振幅 A に依存しない）。
だから K（≈ 0.6073）を一度だけ計算しておけば、どんな入力にも使い回せます。
④
補正は掛け算ひとつ
最終ベクトルの長さ |v| は真の振幅 A より 1/K 倍大きくなっているので：
A = |v| × K
グラフの 緑の点 が K 補正後の点です。 斜辺の先端（白点）から緑点への破線が「縮め戻し」を示しています。
💡 直感的なたとえ： コピー機で原稿を 8 回コピーするたびに 1.08 倍ずつ拡大されてしまうとします。 最終的に何倍になるかは計算できます（1.08⁸ ≈ 1.85 倍）。 なら縮小率 1/1.85 ≈ 0.54 をかければ元のサイズに戻せます。 CORDICのK補正はこれと同じ発想です。

// Verilog / FPGA 実装ガイド

CORDICベクタリングモードは シフト＋加減算のみ で動作するため、乗算器を持たない古典的なFPGAでも実装できます。 入力 (I, Q) からθとAを復元する回路の要点と、実際に合成・動作するVerilogサンプルを示します。

① 固定小数点表現とビット幅
入力 I, Q は signed [15:0]（Q1.15形式 = 1符号ビット＋15小数ビット）。
演算途中でビットが1〜2bit伸びるため、中間レジスタは signed [17:0] 程度に広げるか、 入力時に1〜2bit右シフトして余裕を持たせます。
角度の内部表現: 180° = 2¹⁴ = 16384 とスケーリング。1LSB ≈ 0.011°。
② あらかじめ保持する定数テーブル（ROM）
角度テーブル（ステップ数分だけ必要、起動後は変化なし → Quartus/Vivadoが自動的にLUTやBlock RAMに配置）：
ATAN[i] = arctan(2⁻ⁱ) を固定小数点で保持。
K補正係数（ステップ数が決まれば固定値）：
K = 0.6073... → Q1.15形式 = 19898 (16'sh4DBA)
これらは localparam として宣言するだけで合成ツールが定数扱いします。
③ アーキテクチャ選択：パイプライン vs 逐次処理
パイプライン型（下のサンプル）— N段のフリップフロップを連結。毎クロック1サンプル入力可能。レイテンシ = N_ITER サイクル。高スループットが必要な場合に向く。
逐次処理型（ステートマシン）— 1段を N_ITER 回繰り返す。レイテンシ同じだがスループット 1/N_ITER。小面積が優先の場合に向く。
SDRやFMラジオなどのリアルタイム処理 → パイプライン型推奨。
④ 実装上の注意点
入力象限チェック（事前回転）：I = A·cos(θ) < 0（象限II/III）のとき収束しない。I の符号のみチェックし、I<0なら (I,Q)→(Q,−I)または(−Q,I) に変換してから渡す。θの値は不要。
ビットシフトの符号：>>>（算術右シフト）を使用。論理シフト >> では符号ビットが化ける。
K補正の乗算：K補正だけは乗算が必要。FPGAのDSPブロック（Multiplier）1個で実現可能。乗算を避けたい場合は K≈0.625 = 1/2 + 1/8 と近似してシフト加算で実装できます。
cordic_vectoring.v IQ復調CORDICコア ／ パイプライン実装 ／ 8ステップ ／ 16bit固定小数点 
// ================================================================
// cordic_vectoring.v  —  CORDIC Vectoring Mode (IQ Demodulation)
//
// 入力 : I_in = A·cos(θ),  Q_in = A·sin(θ)   [signed 16-bit Q1.15]
// 出力 : phase     ≈ θ  [固定小数点, 180° = 16384]
//        amplitude ≈ A  [K補正済み, Q1.15]
// 方式 : N_ITER段パイプライン — 毎クロック1サンプル処理
//        レイテンシ = N_ITER + 1 サイクル
// ================================================================
module cordic_vectoring #(
    parameter DW     = 16,   // データビット幅
    parameter N_ITER = 8     // 反復回数（精度）
)(
    input  wire                    clk,
    input  wire                    rst_n,
    input  wire                    valid_i,
    input  wire signed [DW-1:0]  I_in,    // Iサンプル
    input  wire signed [DW-1:0]  Q_in,    // Qサンプル
    output reg                     valid_o,
    output reg  signed [DW-1:0]  phase,   // 推定位相θ
    output reg  signed [DW-1:0]  amplitude // 推定振幅A（K補正後）
);

// ─── ① 定数テーブル（合成時にLUTまたはBlock RAMに配置）─────────
// arctan(2^-i) の固定小数点：スケール 180° = 2^14 = 16384
localparam signed [15:0] ATAN[0:7] = '{
    16'sd4096,  // i=0 : arctan(2^-0) = 45.000°
    16'sd2419,  // i=1 : arctan(2^-1) = 26.565°
    16'sd1278,  // i=2 : arctan(2^-2) = 14.036°
    16'sd 649,  // i=3 : arctan(2^-3) =  7.125°
    16'sd 326,  // i=4 : arctan(2^-4) =  3.576°
    16'sd 163,  // i=5 : arctan(2^-5) =  1.790°
    16'sd  81,  // i=6 : arctan(2^-6) =  0.895°
    16'sd  41   // i=7 : arctan(2^-7) =  0.448°
};
// K補正係数: K ≈ 0.6073  →  Q1.15形式: 0.6073 × 32768 ≈ 19898
localparam signed [DW-1:0] K_FACTOR = 16'sh4DBA;  // = 19898

// ─── ② パイプラインレジスタ（N_ITER + 1段）─────────────────────
reg signed [DW-1:0] xp[0:N_ITER];  // x成分（最終的に振幅/K に相当）
reg signed [DW-1:0] yp[0:N_ITER];  // y成分（0 に収束させる）
reg signed [DW-1:0] zp[0:N_ITER];  // 累積角度（θ に収束）
reg                  vp[0:N_ITER];  // valid信号

// ─── ③ ステージ0 : 入力ラッチ＆象限チェック（事前回転）─────────
// |θ| > 90° のとき I_in < 0  →  先に ±90° 回転して x ≥ 0 を保証
always @(posedge clk) begin
    vp[0] <= valid_i;
    if (!I_in[DW-1]) begin           // I >= 0: |θ| ≤ 90°, そのまま
        xp[0] <= I_in;  yp[0] <= Q_in;  zp[0] <= 0;
    end else if (!Q_in[DW-1]) begin  // I < 0, Q >= 0: 第2象限 → +90°回転
        xp[0] <=  Q_in;  yp[0] <= -I_in;  zp[0] <= 16'sd8192;  // +90°
    end else begin                    // I < 0, Q < 0: 第3象限 → -90°回転
        xp[0] <= -Q_in;  yp[0] <=  I_in;  zp[0] <= -16'sd8192; // -90°
    end
end

// ─── ④ CORDICパイプラインステージ（generate で N_ITER 段展開）────
// 乗算なし : x >>> i  は算術右シフト = x × 2^-i
// 方向判定 : y の符号ビット1つだけ見ればよい（θ を知らなくてよい）
genvar i;
generate
    for (i = 0; i < N_ITER; i = i + 1) begin : stage
        always @(posedge clk) begin
            vp[i+1] <= vp[i];
            if (yp[i] >= 0) begin       // y ≥ 0 → CW回転（yを引き下げる）
                xp[i+1] <= xp[i] + (yp[i] >>> i);
                yp[i+1] <= yp[i] - (xp[i] >>> i);
                zp[i+1] <= zp[i] + ATAN[i];
            end else begin               // y < 0 → CCW回転（yを引き上げる）
                xp[i+1] <= xp[i] - (yp[i] >>> i);
                yp[i+1] <= yp[i] + (xp[i] >>> i);
                zp[i+1] <= zp[i] - ATAN[i];
            end
        end
    end
endgenerate

// ─── ⑤ 出力ステージ : K補正して振幅を復元 ───────────────────────
// xp[N_ITER] ≈ A/K  →  × K_FACTOR(Q1.15) >> 15  で真の A が得られる
// DSPブロック1個分の乗算（FPGAの場合は自動的にDSPスライスに配置）
always @(posedge clk) begin
    valid_o   <= vp[N_ITER];
    phase     <= zp[N_ITER];                              // θ (180°=16384)
    amplitude <= (xp[N_ITER] * K_FACTOR) >>> (DW - 1);  // K補正済み振幅
end

endmodule
📐 角度の読み方： phase 出力をdegreeに変換 → θ[°] = phase × 180.0 / 16384
🔧 ステップ数の変更： N_ITER パラメータを変えるだけ。8→12にすると精度が上がりLUTが増える。
⚡ 合成結果の目安（Xilinx 7-series）： N_ITER=8, DW=16 で約 320 LUT + 1 DSP48E ／ Fmax ≈ 250MHz
💡 K補正を乗算なしで近似： K ≈ 0.625 = 1/2 + 1/8 → (x >> 1) + (x >> 3) で誤差 ≈ 3%
